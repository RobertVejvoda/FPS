## Types of Testing Applied

### Unit Testing
- **Description:** Tests individual components or functions in isolation.
- **Purpose:** Ensures that each component functions correctly on its own.
- **Tools:** JUnit, NUnit, XUnit, pytest.

### Integration Testing
- **Description:** Tests the interaction between integrated components or systems.
- **Purpose:** Ensures that components work together as expected.
- **Tools:** Postman, REST Client, JUnit.

### Functional Testing
- **Description:** Tests the system against functional requirements.
- **Purpose:** Ensures that the system performs its intended functions.
- **Tools:** Selenium, QTP, TestComplete.

### Performance Testing
- **Description:** Tests the system's performance under various conditions.
- **Purpose:** Ensures that the system meets performance criteria.
- **Tools:** JMeter, LoadRunner, Gatling.

### Security Testing
- **Description:** Tests the system's security measures.
- **Purpose:** Ensures that the system is secure from vulnerabilities.
- **Tools:** OWASP ZAP, Snyk, SonarQube, Veracode.

### Usability Testing
- **Description:** Tests the system's user interface and user experience.
- **Purpose:** Ensures that the system is user-friendly and meets user expectations.
- **Tools:** UserTesting, Lookback, Crazy Egg.

### Regression Testing
- **Description:** Tests the system after changes to ensure existing functionality is not broken.
- **Purpose:** Ensures that new changes do not introduce new bugs.
- **Tools:** Selenium, QTP, TestComplete.

### Acceptance Testing
- **Description:** Tests the system against acceptance criteria.
- **Purpose:** Ensures that the system meets the requirements and is ready for production.
- **Tools:** Cucumber, FitNesse, TestRail.
